互联网计算常用口令

常用展示口令

展示路由表

Router#show ip route

展示NAT转换

Router#show ip nat translations

简化 Router#show ip nat tr

展示ACL访问控制

Router#show access-list

展示VLAN配置

Router#show vlan

Router#show vlan brief

展示路由协议

Router#show ip protocol

简化 Router#show ip pr

配置常用口令

  1. 配置动态路由协议OSPF

    • Router(config)#router ospf 1{需要为从1开始的具体数字，表示ospf的进程号}
    • Router(config)#net 192.168.1.0{需要宣告的网段} 0.0.0.255{校验位} area 0{所处区域编号}
    • Router(config)#no net 192.168.1.0{不在宣告的网段} 0.0.0.255{校验位} area 0{所处区域编号}

  2. 配置VLAN以及trunk技术

    • Switch(config)#vlan 10{vlan编号}

    • Switch(config-vlan)#name VLAN10{vlan 10的名字，可以任意取}

  • Switch(config)#int f0/2{进入对应的Switch端口}

  • Switch(config-if)#switchport mode access/trunk{配置端口模式}

    简化为 Switch(config-if)#sw mo acc/tr

  • Switch(config-if)#switchport access vlan 10{access端口模式下，将端口配置向对应的vlan}

    简化为 Switch(config-if)#sw acc vlan 10

    删去VLAN

  • Switch(config)#no vlan 10{编号}

    注：直接删除vlan会使得该vlan原先匹配的端口无法连通，一般先将该vlan的端口转移至默认vlan 1，再作删除操作

  1. 配置单臂路由

    配置子网前建议先将原主网打开(Router(config)#int f0/0 Router(config-if)#no shut)

    单臂路由对应的switchport应该设置为trunk模式

    • Router(config)#int f0/0.1 {进入f0/0的子网段}

    • Router(config-subif)#en dot1q 10{使用dot1q协议封装协议报文至vlan 10，指令中dot1q后面是多少就封装到哪个网段}

      之后配置ip地址和子网掩码，对应网段的主机的默认网关应为路由器中的对应ip地址

  2. 配置三层交换机

    需要在三层交换机中创建vtp域并设置当前交换机在vtp的工作模式为server，创建vlan、配置vlan地址分配并在设置端口为access模式后设置端口为trunk模式。此外，还需打开交换机路由功能

    Switch(config)#vtp domain test{创建vtp域}

    Switch(config)#vtp mode server(设置当前交换机在vtp的工作模式为server)

    Switch(config)#vlan 10{创建vlan}

Switch(config)#interface f0/1{进入端口}

Switch(config-if)#sw mo acc{修改端口模式为access}

注：因为cisco的三层交换机有一个auto模式，不能直接配置trunk

Switch(config-if)#sw mo tr{修改端口模式为trunk}

Switch(config)#ip routing{启动路由}

Switch(config)#int vlan 10{进入vlan 10}

Switch(config-if)#ip add 192.168.1.1 255.255.255.0 {为vlan 10配置地址}

需要在第二层交换机中配置vtp中客户机模式，设置对应trunk端口，并将对应主机划分入vlan中

  1. 配置ACL访问列表

    • Router(config)#access-list 123{访问列表编号} deny/permit icmp{报文类型} host 192.168.6.2{主机的ip地址} 192.168.7.0 0.0.0.255{禁止/允许访问的网段}

    • Router(config)#no access-list 123{删去的访问列表编号}

      注：Router(config)#access-list 123 deny/permit icmp any any

      {访问列表默认匹配为拒绝，需要在结尾增加permit any any允许报文传递}

    • Router(config-if)#ip access-group 123{使用的访问列表的编号} in/out{配置进/出端口}

      标准访问控制列表 ：Router(config)#access-list 10 deny host 192.168.6.2
      根据数据包的源IP地址来允许或拒绝数据包。
      访问控制列表号是1-99以及1300~1999。

      扩展访问控制列表：Router(config)#access-list 123 deny icmp host 192.168.6.2 192.168.7.0 0.0.0.255
      根据数据包的源IP地址、目的IP地址、指定协议、端口和标志来允许或拒绝数据包。
      访问控制列表号是100-199以及2000~2699。

  2. 配置静态NAT

    • Router(config)#ip nat inside source static 10.0.1.2 192.168.8.2
    • Router(config-if)#ip nat inside/outside

  3. 配置动态NAT

  4. 配置PAT

    • 第一步，设置外部端口。

      interface f0/1

      ip address 202.99.160.1 255.255.255.0

      in nat outside

    • 第二步，设置内部端口。

      interface f0/0

      ip address 10.100.100.1 255.255.255.0

      ip nat inside

    • 第三步，定义合法IP地址池。

      in nat pool onlyone 202.99.160.2 202.99.160.2 netmask 255.255.255.252

      // 指明地址缓冲池的名称为onlyone,IP地址范围为202.99.160.2,子网掩码为255.255.255.252。由于本例只有一个IP地址可用，所以，起始IP地址与终止IP地址均为202.99.160.2。如果有多个IP地址，则应当分别键入起止的IP直址。

    • 第四步，定义内部访问列。

      access-list 1 permit 10.100.100.0 0.0.0.255

      允许访问Internetr的网段为10.100.100.0~10.100.100.255，子网掩码为255.255.255.0。需要注意的是，在这里子网掩码的顺序跟平常所写的顺序相反，即0.255.255.255。

    • 第五步，设置复用动态地址转换。

      在全局设置模式下，设置在内部的本地地址与内部合法IP地址间建立复用动态地址转换。命令语法如下：ip nat inside source list访问列表号pool内部合法地址池名字overload

一些概念

  1. 静态NAT、动态NAT、PAT端口多路复用的区别

    静态转换（static translation）将内部网络的私有IP地址转换为公有合法IP地址。IP地址的对应关系是一对一的，而且是不变的。借助于静态转换，可以实现外部网络对内部网络中某些特定设备(如服务器)的访问。

    动态转换（dynamic translation）指将内部私有IP转换为公网IP地址时，IP的对应关系是不确定的。也就是说只要指定哪些内部地址可以进行NAT转换，以及哪些可以的合法的IP地址可以作为外部地址，就可以进行动态转换了。也可以使用多个合法地址集。当ISP提供的合法IP地址略少于网络内部的计算机数量时。可以采用动态转换的方式。

    端口多路复用（port address translation，）PAT 改变外出数据包的源IP地址和源端口并进行端口转换，即端口地址转换采用端口多路复用的方式。内部网络的所有主机均可共享一个合法外部IP地址实现互联网的访问。最大程度上节约IP地址资源。

    端口多路复用使用的是端口号的转换，也是不确定的，所以不能和动态转换同时使用，这个问题可以通过实施静态NAT映射来避免。

    NAT端口映射关系：是将TCP或UDP协议中的内部局部地址需要转换的端口号转换成为内部全局地址的端口号。

  2. RIP和OSPF的区别

    1. 【名字不同】

      • RIP（路由信息协议）——分布式的基于距离向量的路由选择协议
      • OSPF（开放最短路径优先协议）—— 使用分布式的基于链路状态的路由选择协议

    2. 【工作核心不同】

      • RIP：数跳数
      • OSPF：计算链路的度量值

    3. 【向谁发】

      • RIP：仅和相邻路由器交换信息；
      • OSPF：向本自治系统所有路由器发送消息，由于路由器发送的链路状态信息只能单向传送，OSPF不存在“坏消息传播得慢”的问题，更新过程的收敛性得到保证。

      【发什么】

      • RIP：路由器交换的信息是当前本路由器所知道的全部信息，即自己现在的路由表
      • OSPF：发送的信息是与本路由器相邻的所有路由器的链路状态 ，只涉及与相邻路由器的连通状态，与整个互联网的规模无关。

      【什么时候发】

      • RIP：按固定的时间间隔交换路由信息（当网络拓扑发生变化时，路由器也及时向相邻路由器通告拓扑变化后的路由信息）
      • OSPF：在网络刚刚启动计算第一次路由表时，一定发路由信息。只有当链路状态发生变化时，路由器才能向所有路由器用洪泛法发送此消息
      • RIP协议使用运输层的用户数据包UDP来进行传送
      • OSPF的位置在网络层，直接用IP数据报传送(其IP数据报首部的协议字段值为89)。由于OSPF构成的数据报很短，不仅减少了路由信息的通信量，而且在传送中不必分片，不会出现一片丢失而重传整个数据报的现象。
      • RIP：不能在两个网络之间同时使用多条路由，选择一条具有最少路由器的路由即最短路由
      • OSPF：如果到同一个目的网络有多条相同代价的路径时，可以将通信量分配给这几条路径，做到路径间的负载平衡
      • RIP：限制了网络规模，能使用的最大距离为15,16表示不可达
      • OSPF：链路的度量可以是1~65 535中的任何一个无量纲的数，可供管理人员来决定。因此十分灵活。
      • RIP：1号版本不支持子网划分，2号版本支持子网划分
      • OSPF在路由分组中包含子网掩码，支持可变长度的子网划分和无分类的编址CIDR

    8. 所有在OSPF路由器之间交换的分组（如链路状态更新分组）都具有鉴别功能，因而保证了仅在可信赖的路由器之间交换链路状态信息。

    9. 由于各路由器之间频繁地交换链路状态信息，因此所有的路由器最终都能建立一个链路状态数据库，及即全网拓扑结构图。

      • OSPF的链路数据库能较快地进行更新，使每个路由器能及时更新其路由表，OSPF的更新过程收敛得快是其重要优点。
      • RIP协议的每个路由器虽然知道到所有的网络距离以及下一跳路由器，但是不知道全网的拓扑结构，只有到了下一跳路由器，才能知道再下一跳应当怎样走

    10. 为了使 OSPF 能够用于规模很大的网络，OSPF 将一个自治系统再划分为若干个更小的范围，叫作区域。利用洪泛法交换链路状态信息的范围局限于每一个区域而不是整个的自治系统，减少了整个网络上的通信量，减轻路由器的负担。

  1. 关于access-list

    • 技术原理：

      ACLs的全称为接入控制列表（Access Control Lists），也称为访问列表（Access List），俗称为防火墙，在有的文档中还称之为包过滤。ACLs通过定义一些规则对网络设备接口上的数据报文进行控制：允许通过或丢弃，从而提高网络可管理性和安全性；

      IP ACL分为两种：标准IP访问列表和扩展IP访问列表，编号范围分别为199、13001999，100199、20002699；

      标准IP访问列表可以根据数据包的源IP地址定义规则，进行数据包的过滤；

      扩展IP访问列表可以根据数据包的源IP、目的IP、源端口、目的端口、协议来定义规则，进行数据包的过滤；

      IP ACL基于接口进行规则的应用，分为：入栈应用和出栈应用；

    • 一些特点：

      1. 定义permit（允许）deny（拒绝）
      2. 序列号由小到大
      3. 满足金字塔
      4. 一旦匹配，立即执行
      5. 调用是在接口上，in（先看ACL，后看路由），out（先看路由表，再看ACL）
      6. 标准ACL尽量靠近目标，是因为标准的ACL只匹配源IP地址，扩展ACL尽量控制源，但不能在源上（ACL不能过滤自身所产生的流量），因为扩展ACL不仅可以匹配源目IP地址，还可以匹配协议和端口号
      7. 末尾隐含拒绝所有

    • 分类：

      • 标准ACL：仅仅匹配源IP
        • 编号：1-99、1300-1999
        • 命名方式：standard
      • 扩展ACL：匹配源IP，目标IP地址，协议，端口
        • 编号：100-1299、2000-2699
        • 命名方式：extended

  2. 单臂路由和三层交换机

    三层交换机就是具有部分路由器功能的交换机，三层交换机的最重要目的是加快大型局域网内部的数据交换，所具有的路由功能也是为这目的服务的，能够做到一次路由，多次转发。对于数据包转发等规律性的过程由硬件高速实现，而像路由信息更新、路由表维护、路由计算、路由确定等功能，由软件实现。三层交换技术就是二层交换技术+三层转发技术。

    三层交换机 需要调用指令 Switch(config)#ip routing 开启路由功能

    由于单臂路由依旧是个路由器，在数据转发方面始终不如三层交换机，三层交换机可以高速转发数据以及不一样数据之间的交换，因此，当电脑数量不多的时候，运用单臂路由就足够了，但是，如果局域网电脑速度比较多，最好选择三层交换机，这样就不会影响局域网电脑网络传输速率
